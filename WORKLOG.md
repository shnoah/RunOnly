### 상세 화면 1차 로딩 최적화
- 러닝 상세 첫 진입 시 route / 심박 존 / 미사용 고급 러닝 메트릭까지 한 번에 읽던 구조를 정리.
- 현재는 기본 로딩에서 다음 데이터만 먼저 읽도록 조정:
  - 거리 샘플
  - 심박
  - 걸음 수
- route는 거리 샘플 fallback이 필요할 때만 즉시 읽고, 일반적인 경우에는 상세 화면이 뜬 뒤 보강 로딩하도록 변경.
- 심박 존 프로필도 기본 로딩 이후 보강 로딩으로 이동.
- 상세 화면 하단의 `러닝 경로`, `심박 존 1-5` 섹션은 보강 로딩 중일 때 즉시 `데이터 없음`으로 보이지 않도록 로딩 문구를 표시.
- 상세 화면 컨테이너는 `LazyVStack`으로 바꿔 오프스크린 섹션 생성 비용도 약하게 줄임.

### 상세 후처리 매칭 비용 최적화
- 심박/케이던스/추가 메트릭을 거리 타임라인에 붙일 때, 매 샘플마다 전체 타임라인을 `filter + min`으로 훑던 경로를 정리.
- 현재는 세그먼트별 타임라인 lookup을 한 번 만든 뒤, 각 샘플은 날짜 기준 이진 탐색으로 가장 가까운 포인트를 찾도록 변경.
- 긴 러닝에서 상세 후처리 CPU 비용이 커지는 구간을 줄이는 목적의 2차 최적화.

### 공유 편집 화면 압축

### 스티커 템플릿 재설계
- 스티커 캔버스를 더 좁은 비율로 줄여 `포스터`보다 `스티커`에 가까운 비례로 조정.
- 스티커 기본 포함 데이터는 `경로 + 거리 + 시간 + 페이스`로 정리하고, 날짜는 기본값에서 제외.
- 로고와 루트를 상단으로 올리고, 값은 가운데 정렬 기준으로 아래에 한 줄씩 쌓이도록 재배치.
- 경로는 시작점/종료점 마커 없이 선만 남기고, 로고와 루트 강조색은 앱에서 자주 쓰는 초록 계열로 통일.
- 메트릭 라벨(`거리`, `시간`, `페이스`)과 검정 반투명 칩 배경은 제거.
- `경로만 선택`했을 때 보이던 `RUN SHARE` 플레이스홀더 문구도 제거.

### 현재 스티커 방향
- 위에서부터 `RUNONLY -> route -> distance/time/pace` 순서의 세로 흐름을 사용.
- 값은 예시처럼 한 줄당 하나씩 크게 표기:
  - `4.76 km`
  - `28:57`
  - `06:04/km`
- 경로 폭에 맞춰 스티커 캔버스 가로폭도 함께 줄여 중앙 정렬 느낌을 강화.
