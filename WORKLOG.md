### 현재 스레드 작업 내역

#### 1. 상세 거리/그래프/스플릿 불일치 수정
- 상세 카드의 총 거리와 그래프/스플릿 기준이 서로 다르던 문제 확인.
- 초기 단계에서는 `HKWorkout.totalDistance`를 기준으로 route 거리축과 split 계산을 맞추는 방향으로 조정.
- 그래프 축이 `4.0`에서 끊기지 않고 실제 최대 거리(`4.02`)까지 표기되도록 수정.
- 마지막 partial split도 표시되도록 수정.

#### 2. 그래프 해상도 조정
- 거리 버킷(대략 200m) 기반 압축을 제거.
- 샘플을 더 촘촘히 사용하도록 변경해 실주행 그래프 디테일 향상.

#### 3. Apple 공식 기록에 더 가깝게 상세 계산 구조 변경
- 목표를 `재계산 앱`이 아니라 `공식 운동 기록을 보기 좋게 표시하는 앱`으로 재정의.
- route를 지도 표시용으로 두고, 상세 계산의 기준을 다음으로 전환:
  - `HKWorkout.totalDistance`
  - `HKWorkout.duration`
  - `HKWorkout.workoutEvents`(pause/resume, motion pause/resume)
  - `HKQuantityTypeIdentifierDistanceWalkingRunning` 샘플
- `RunDetail`에 `distanceTimeline` 모델 추가.
- pace/split/선택 지점 elapsed 계산을 route 재계산 대신 공식 거리 타임라인 기준으로 변경.
- distance 샘플이 없을 때만 route 기반으로 fallback하도록 구성.

#### 4. pause 구간 반영
- active interval 개념 도입.
- pause 또는 auto-pause 구간은 active interval 밖으로 분리.
- 그래프가 pause 전후를 하나의 연속선으로 잇지 않도록 segment 기반 시리즈로 분리.
- pace smoothing도 segment 경계를 넘지 않게 수정.

#### 5. split UI 조정
- split 행 우측 하단의 중복 시간 표시 제거.
- 해당 위치에는 split 평균 심박이 나오도록 변경.
- 현재 사용자 확인 기준:
  - 그래프 pause 처리: 대체로 정상처럼 보임.
  - split 평균 심박 표기: 문제 남음, 다음 작업으로 보류.

- 공유 편집 시트에서 다음 기능을 제공:
  - 템플릿 선택: `스티커 / 정사각형 / 스토리`
  - 포함할 데이터 토글: 경로, 날짜, 환경, 거리, 시간, 페이스, 심박, 케이던스, 신발
  - 실시간 미리보기
- `스티커` 템플릿은 투명 배경 PNG를 전제로 한 레이아웃으로 구성.
- 경로는 지도 스냅샷 대신 route polyline을 직접 렌더링하는 방식으로 구현.
- 내보내기는 다음 두 경로를 지원:
  - PNG를 클립보드에 복사
  - PNG 파일을 공유 시트로 전달
- 디자인 방향은 최근 Strava `Sticker Stats`류의 `투명 오버레이 + 경로 + 메트릭 칩` 구성을 참고해 반영.
- 실제 소셜 앱별 투명도 처리 차이는 기기에서 추가 검증이 필요함.

### 빌드/검증 메모
- `DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer`와 임시 `DerivedData` 경로를 사용하면 CLI 빌드가 Swift 컴파일 단계까지는 진행됨.
- 현재 최종 실패는 코드 문법 에러가 아니라 asset catalog 단계의 환경 문제로 확인.
- 현재 관찰되는 환경 에러:
  - `No available simulator runtimes for platform iphonesimulator`

## 2026-03-12 (공유 이미지 다듬기)

### 공유 편집 화면 압축
- 공유 시트의 편집 화면을 `미리보기 + 템플릿 + 포함 데이터`가 한 화면에서 보이도록 재구성.
- 하단 액션은 `저장 / 복사 / 공유` 1줄 버튼으로 압축.
- 템플릿별 미리보기 폭을 따로 조절해 스티커가 지나치게 크게 보이지 않도록 정리.

### 저장/권한 보강
- 공유용 PNG를 카메라롤에 저장할 수 있도록 사진 추가 권한 문구를 `Info.plist`에 추가.
- 공유 편집 화면에서 `사진 저장` 버튼으로 바로 저장 가능하게 연결.

### 스티커 템플릿 재설계
- 스티커 캔버스를 더 좁은 비율로 줄여 `포스터`보다 `스티커`에 가까운 비례로 조정.
- 스티커 기본 포함 데이터는 `경로 + 거리 + 시간 + 페이스`로 정리하고, 날짜는 기본값에서 제외.
- 로고와 루트를 상단으로 올리고, 값은 가운데 정렬 기준으로 아래에 한 줄씩 쌓이도록 재배치.
- 경로는 시작점/종료점 마커 없이 선만 남기고, 로고와 루트 강조색은 앱에서 자주 쓰는 초록 계열로 통일.
- 메트릭 라벨(`거리`, `시간`, `페이스`)과 검정 반투명 칩 배경은 제거.
- `경로만 선택`했을 때 보이던 `RUN SHARE` 플레이스홀더 문구도 제거.

### 현재 스티커 방향
- 위에서부터 `RUNONLY -> route -> distance/time/pace` 순서의 세로 흐름을 사용.
- 값은 예시처럼 한 줄당 하나씩 크게 표기:
  - `4.76 km`
  - `28:57`
  - `06:04/km`
- 경로 폭에 맞춰 스티커 캔버스 가로폭도 함께 줄여 중앙 정렬 느낌을 강화.

### 검증 메모
- CLI 기준 `ContentView.swift`, `RunningWorkoutsViewModel.swift`, `HealthKitService.swift`의 신규 Swift 컴파일 에러는 보이지 않음.

## 2026-03-14 (공유 이미지 날씨 재추가 / 라벨 확대)

### 공유 이미지 날씨 재추가
- 공유 이미지 옵션에 `날씨`를 다시 추가하고 기본 포함 항목에 넣음.
- 표기는 `섭씨 + 한글 상태` 형식으로 정리:
  - 예: `8°C 맑음`
- 현재는 route 대표 좌표와 러닝 시작 시각 기준으로 Open-Meteo archive API를 조회.
- 직전 실패 원인이었던 미래 `end_date` 포함 문제를 막기 위해 요청 날짜를 오늘 이하로 clamp.

### 메트릭 라벨 가독성 조정
- 거리/시간/페이스/심박/케이던스 라벨 글자 크기를 크게 조정.
- 기존 스티커 구조와 배치는 유지하고 라벨 가독성만 보강.

### 검증 메모
- `swiftc -typecheck RunOnly/*.swift` 통과.
- `xcodebuild`는 Swift 컴파일 단계까지 진행되며, 최종 실패는 기존과 동일한 asset catalog / simulator runtime 환경 문제.
