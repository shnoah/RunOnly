- 지원 화면에서는 내부용 `앱 리뷰 노트 초안` 링크도 제거했다.
- 샘플 러닝은 deprecated `HKWorkout` initializer 없이 표시용 값만 보관하도록 분리했고, 실제 HealthKit 러닝일 때만 원본 `HKWorkout`를 들고 있게 정리했다.
- 원본 workout 참조가 없는 경우 상세 데이터와 경로 로딩은 명확한 오류로 빠지게 안전장치를 추가했다.

### UI와 정보 구조 정리
- 상세 화면을 `상단 요약 → 경로 → 러닝 흐름 → 구간 기록 → 심박 분포 → 러닝화 → 원본 안내` 순서로 재배치했다.
- 상단 요약은 숫자판과 긴 문장 중심에서 히어로 카드 구조로 바꾼 뒤, 최종적으로는 `거리 / 시간 / 페이스 / 심박 / 케이던스 / 상승`만 바로 읽히는 값 중심 타일로 더 압축했다.
- 하단 원본 표기는 bundle id 노출 대신 `Apple 건강에 저장된 러닝 기록` 안내 성격으로 바꾸고, 섹션 제목과 보조 문구도 더 짧게 다듬었다.

- `RunDetail`에 `distanceTimeline` 모델을 추가했고, pace, split, 선택 지점 elapsed 계산은 공식 거리 타임라인 기준으로 바꾸고 distance 샘플이 없을 때만 route fallback을 쓰게 했다.
- pause와 auto-pause를 active interval 밖으로 분리하고, 그래프는 pause 전후를 하나의 연속선으로 잇지 않도록 segment 기반 시리즈로 나눴으며, pace smoothing도 segment 경계를 넘지 않게 했다.
- split 행 우측 하단의 중복 시간 표기는 제거하고 그 자리에 split 평균 심박을 넣었다.
- 이 단계의 주요 수정 파일은 `RunOnly/HealthKitService.swift`, `RunOnly/RunningWorkout.swift`, `RunOnly/ContentView.swift`였다.

### 남은 이슈
- 당시 남은 문제는 split 평균 심박 표기였다.
- 원인 후보로는 split 거리 구간과 heart rate 샘플 매핑 차이, pause 경계 전후 샘플 필터링 문제, 특정 러닝의 heart rate 샘플 부족 또는 distance 매핑 오차를 봤다.
- pause 처리 자체는 대체로 정상처럼 보였고, 다음 단계 과제로는 split 평균 심박 버그 수정과 필요 시 Apple Fitness 비교용 디버그 출력이나 진단 화면 추가가 남아 있었다.
