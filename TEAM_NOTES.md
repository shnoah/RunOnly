# Team Notes

## Collaboration Preferences

- 검증 결과를 정리할 때는 핵심 결과만 간단히 적고, 환경 이슈는 사용자가 따로 요청할 때만 언급한다.
- 작업 내용 기록은 `WORKLOG.md`에 남기고, 협업 방식이나 선호 같은 메모는 이 파일에 이어서 적는다.

## UI Global Rules

- UI는 항상 트렌디해야 한다.
- UI는 항상 간결해야 한다.
- UI는 항상 직관적이어야 한다.
- 처음 보는 사용자도 한눈에 이해할 수 있어야 한다.
- 헤더/카드 UI를 바꿀 때 위 원칙을 우선 적용한다.
